Flutter Earth

A modern desktop tool for downloading and processing satellite imagery using Google Earth Engine

Features

  • Modern Electron Interface: Sleek, responsive UI for a seamless user experience.
  • Modular Python Backend: Professional Python package for maintainability and scalability.
  • Theming Support: Switch between 'Dark', 'Light', and custom themes.
  • Advanced Downloading: Download imagery from multiple sensors (Landsat, Sentinel-2, etc.).
  • On-the-fly Processing: Includes cloud masking and pre-processing capabilities.
  • Robust Configuration: Class-based configuration system manages all settings.
  • Comprehensive Satellite Info: Browse detailed information about available satellites.
  • Offline Mode: Use the app with limited features when Earth Engine credentials are missing.

Prerequisites & Installation

  • Python 3.8 or higher
  • Node.js 18+ (for Electron frontend)
  • Google Earth Engine account (for full functionality)

Clone the repository:

git clone https://github.com/yourusername/flutter-earth.git
cd flutter-earth

Install Python dependencies:

pip install -r requirements.txt

Install frontend dependencies:

cd frontend
npm install

Run the app:

cd ..
run_desktop.bat

Usage

  • Start the app and sign in with your Google Earth Engine account if prompted.
  • Select your area of interest and satellite data source.
  • Configure download and processing options.
  • Download and process imagery directly to your desktop.
  • Browse satellite info and manage your settings.

How it Works

Flutter Earth uses a modern Electron-based frontend for the user interface and a Python backend for all satellite data processing. The backend communicates with Google Earth Engine, handles authentication, and manages downloads and processing tasks. The frontend and backend communicate via a local API, providing a seamless desktop experience.

FAQ

Q: Do I need a Google Earth Engine account?
A: Yes, for full functionality. Offline mode is available with limited features.
Q: Can I use this on Mac or Linux?
A: The app is designed for Windows, but may work on other platforms with minor adjustments.
Q: Where are downloads saved?
A: By default, downloads are saved to the application's working directory. You can change this in the settings.
Q: How do I update the app?
A: Pull the latest changes from GitHub and re-run npm install and pip install -r requirements.txt if needed.
